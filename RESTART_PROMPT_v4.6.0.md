# ParleyMap v4.6.0 recovery summary

Use this build if the thread or working directory is lost.

The accepted layout is anchored: top-10 opening map, selected profile on the right, and intelligence modules below the map. v4.6.0 keeps that structure and fixes the most recent usability issues:

- Network maps no longer pile up repeated appearances as separate face cards. They aggregate repeated activity into city heat bubbles and keep one face marker per person.
- Influence Timeline wraps vertically to show more content without a cramped horizontal strip.
- Relationship web keeps Open People on the right and places the selected source-backed event layer below the chart.

Deployment remains GitHub Pages from repo root. The root must contain index.html, CNAME, .nojekyll, data/, assets/, src/, templates/, scripts/, docs/, impressum.html, privacy.html, and corrections.html.
