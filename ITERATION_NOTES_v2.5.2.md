# Visual audit queue

The preview build supports face cards, but production visuals must be treated as unapproved until rights metadata is captured. This queue shows what the v0.4.2 demo knows today.

Approval fields still needed for production: source page, file URL, author, license identifier, license URL, attribution text, public-domain or compatible-license decision, modification notes, retrieval timestamp, reviewer, and takedown path.
