ParleyMap rescue context for a new assistant thread

Repo: github.com/Goaskbid/parleymap
Live site: parleymap.com

Critical facts:
- The live dataset is embedded inside index.html in <script id="demo-data" type="application/json">.
- Never overwrite the whole index.html from a partial JSON file.
- Current repo history contained a bad roster auto-update that expanded people from 94 to 137 and introduced historical active leaders such as Vincent Auriol.
- Current live screens showed fake dated cards such as City of London finance diplomacy watch, IAEA nuclear diplomacy watch, think-tank watch, and royal diary watch.
- AdSense setup likely already had real ca-pub and data-ad-slot values in a previous index.html. Do not invent or replace those values.

This package is meant to fix both:
1. Data integrity.
2. AdSense preservation and ads.txt.

Key scripts:
- scripts/parleymap-integrity-rescue.mjs
  Restores last safe index.html from git history if current index is thin/polluted, removes fake event/watch rows, fixes critical anchors/faces, demotes historical active holders, installs runtime Leaflet guard.
- scripts/parleymap-adsense-preserve-audit.mjs
  Preserves existing AdSense IDs, restores them from git history if needed, creates ads.txt, fails if it cannot find real IDs.
- scripts/parleymap-final-hard-audit.mjs
  Fails the workflow before commit if people count is unsafe, fake watch rows remain, critical anchors are missing, or AdSense is inconsistent.
- scripts/parleymap-current-holder-safe-review.mjs
  Writes current-holder review diagnostics only. Does not auto-mutate roster.
