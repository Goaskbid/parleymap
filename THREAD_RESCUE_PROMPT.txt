ParleyMap rescue context

The site suffered from two independent problems:

1. Data integrity / map integrity:
   - index.html on main became crawler-visible as a one-line placeholder/file listing instead of the real app shell.
   - data/demo.json still contained duplicate Rafael Grossi rows and fake dated watch cards.
   - visible anchors for Sheinbaum, Pope Leo XIV, Prabowo Subianto and Rafael Grossi were wrong.
   - previous roster auto-update polluted people from 94 to 137 through historical office-holder chains.

2. AdSense/site-surface integrity:
   - ads.txt was missing.
   - crawler-visible HTML did not expose google-adsense-account, adsbygoogle, pagead2.googlesyndication.com or data-ad-slot values.
   - privacy.html and impressum.html were not clean human/legal pages.

This package replaces the existing ParleyMap data validation workflow so the visible Actions entry can be used. It restores a safe full index.html from git history, repairs embedded demo-data only, removes fake watch rows, repairs critical anchors/faces, preserves AdSense IDs from current files/history, creates ads.txt, cleans legal pages, and writes audits.

First workflow to run after upload:

Actions > ParleyMap data validation > Run workflow

If AdSense IDs are not recoverable from history, rerun with workflow inputs:

publisher_id
header_slot_id
sidebar_slot_id
